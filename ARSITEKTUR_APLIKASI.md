# 🏗️ ARSITEKTUR APLIKASI ZONASI-HD
**Analisis Mendalam & Desain Teknis Profesional**

## 1. Filosofi Arsitektur (Mengapa Mirip CKG Malimpung?)
Aplikasi CKG Malimpung sukses karena:
- **Frontend-heavy** dengan logic cerdas di client (smart form, OCR hybrid)
- **Firebase** untuk auth + realtime database (cocok untuk kolaborasi tim medis)
- **PWA** untuk ketahanan di lingkungan klinis dengan jaringan tidak stabil
- **Visual & Workflow-driven** (alur pos terintegrasi)

ZONASI-HD mengadopsi **persis pola yang sama** tetapi dengan domain berbeda:
- Bukan skrining umum → **Manajemen risiko cairan HD spesifik**
- Bukan multi-pos → **Single powerful Command Center + Session Workflow**
- Fokus utama: **Kalkulasi klinis akurat + Visualisasi triase warna real-time + Protokol standar**

**Prinsip Desain**:
- **Zero Human Error** pada perhitungan IDWG (pure math function + TypeScript strict)
- **Instant Feedback** (warna muncul < 100ms setelah input berat)
- **High Situational Awareness** di nurse station (dashboard seperti "command center" IGD)
- **Low Cognitive Load** untuk perawat (besar, jelas, warna kuat, aksi jelas per zona)
- **Audit-ready** (setiap sesi tercatat lengkap + timestamp + user)

## 2. Tech Stack Detail & Justifikasi Medis

### Frontend Core
- **React 19 + Vite + TypeScript**: 
  - React 19 hooks terbaru (use, useOptimistic untuk UI responsif)
  - TypeScript: Mencegah bug pada rumus medis (misal berat negatif, division by zero)
  - Vite: HMR super cepat untuk iterasi development klinis

- **Tailwind CSS 4 + Plugin Typography**:
  - Healthcare theme: Background putih/abu klinis (#f8fafc), accent teal medis (#0f766e)
  - Zona warna **sangat prominent** (gunakan `bg-red-600 text-white` dengan shadow untuk urgency)
  - Dark mode opsional (untuk shift malam), tapi default light high-contrast

- **React Router v7 (Data Router)**: Nested routes untuk Dashboard (utama) + detail pasien modal/page

### State Management & Realtime
- **Firebase Cloud Firestore** (bukan Realtime DB):
  - `onSnapshot` untuk dashboard live update ketika perawat lain input data
  - Subcollections: `patients/{id}/sessions/{sessionId}` — natural untuk riwayat
  - Denormalisasi pintar: `patients` doc punya field `latestSession` (zone, pct, date) untuk query dashboard cepat

- **Tidak pakai Redux/Zustand dulu** (kecuali kompleksitas tinggi nanti):
  - Cukup React Context + custom hooks (`useRealtimeDashboard`, `usePatientDetail`)
  - Firebase hooks pattern (mirip CKG)

### Data & Logic Klinis (Inti Aplikasi)
**File kunci**: `src/utils/zonasiCalculator.ts` (pure functions, 100% testable)

export const calculateIDWG = (preHD: number, dryWeight: number): number => {
  if (dryWeight <= 0) throw new Error("BB Kering harus > 0");
  const idwg = ((preHD - dryWeight) / dryWeight) * 100;
  return Math.round(idwg * 10) / 10; // 1 desimal
};

### PWA & Offline

## 3. Struktur Data Firestore (High-Level)

Lihat file **`MODEL_DATA_DAN_FIRESTORE.md`** untuk skema lengkap + rules + indexing.

Ringkas:
- `users/{uid}` — role (PK_II, PK_III, DOKTER, ADMIN), nama, shift
- `patients/{patientId}` — rm, nama, tgl_lahir, bb_kering, latest_idwg_pct, latest_zone, latest_session_date, yellow_streak_count, notes
- `patients/{patientId}/sessions/{autoId}` — date, pre_weight, idwg_pct, zone, nurse_uid, interventions (array), post_weight (opsional), notes
- `alerts/{id}` — untuk Early Warning (generated by logic atau cloud function sederhana)

**Pattern penting**: Setiap write session baru → Cloud Function / client transaction update `patients` doc dengan latest + hitung streak.

Untuk MVP: Logic streak di client saat save (query last 5 sessions → update patient doc).

## 4. User Flow Utama (Mirip Alur Pos di CKG)

1. **Login** (role-based dashboard berbeda sedikit)
2. **Command Center Dashboard** (default landing)
   - Stats ringkasan hari ini (Jumlah pasien per zona + % compliance target)
   - Grid / Table pasien aktif hari ini dengan **kartu warna besar**
   - Klik kartu → Modal Detail Pasien (tren + form input sesi baru cepat)
3. **Input Sesi Baru** (dari dashboard atau halaman pasien)
   - Pilih pasien (searchable combobox)
   - Input BB Pre-HD (number input besar, autofocus)
   - **Instant preview**: IDWG % besar + **Zone Badge raksasa** berubah warna real-time + rekomendasi singkat
   - Tombol "Simpan & Terapkan Protokol"
   - Jika Merah → langsung tampilkan ProtocolModal dengan checklist tindakan PK III
4. **Early Warning Banner** (jika ada pasien dengan streak Kuning ≥3)
5. **Laporan & Kartu** (tab terpisah)
